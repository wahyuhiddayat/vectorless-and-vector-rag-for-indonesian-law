=== TUJUAN TIPE QUERY ===

Tipe `adversarial` menguji apakah sistem retrieval dapat memilih node target yang BENAR ketika kata-kata permukaan query lebih cocok ke node DISTRACTOR. Ini menstress-test reranking (cross-encoder atau LLM rerank), karena BM25 alone akan memilih distractor.

=== INSTRUKSI UTAMA ===

Buat tepat {N} pertanyaan ADVERSARIAL dalam bahasa Indonesia. Setiap query punya:
- Satu anchor TARGET (jawaban benar) di leaf node dokumen ini.
- Satu atau lebih leaf DISTRACTOR di dokumen lain (atau pasal lain dalam dokumen ini) yang punya OVERLAP LEKSIKAL TINGGI dengan query tetapi semantik-nya BUKAN jawaban.

STRATEGI GENERATION:
1. Pilih anchor TARGET di dokumen ini.
2. Identifikasi 1-2 leaf di dokumen lain yang topiknya MIRIP secara permukaan (misalnya: target tentang "hierarki peraturan" di UU 12/2011, distractor tentang "hierarki pengadilan" di PERMA tertentu).
3. Tulis query yang SURFACE TERMS-nya bias ke distractor, tetapi SEMANTIK MENGARAH ke target. Gunakan kata-kata yang muncul di distractor (atau sinonim-nya) dan abstraksi yang menutupi kata-kata literal target.
4. Validator deterministik akan run BM25 atas query, target HARUS tidak masuk top-3, tetapi HARUS masuk top-50. Ini memastikan query benar-benar misleading tetapi target masih reasonable.

POLA YANG BAIK:
- Synonym overlap: target pakai istilah teknis A, distractor pakai istilah teknis B (synonym A), query pakai B yang ada di distractor.
- Topical mislead: target dan distractor sama-sama tentang "kepala daerah" (lexical match), tetapi target tentang "honorarium" sedangkan distractor tentang "pengangkatan". Query menyoal "honorarium" dengan sinonim plus overshadowing kata "pengangkatan".

POLA YANG DILARANG:
- Query yang ambigu, harus tetap ada SATU jawaban yang correct (semantik). Reviewer manusia harus setuju target adalah jawaban yang benar.
- Query yang absurd atau tidak natural. Tetap harus pertanyaan yang masuk akal di domain hukum.
- Query yang terlalu spesifik sehingga hanya match target (gak adversarial lagi).

ATURAN UMUM:
- Self-contained, single-hop, anchor di leaf paling spesifik.
- Hanya body text.
- Adversarial WAJIB ada justifikasi distractor di output (field `distractor_node_id`).

=== REFERENCE MODE ===

Mayoritas adversarial seharusnya `none` atau `doc_only`. Hindari legal_ref (Pasal X eksplisit) karena itu trivializes retrieval.

=== QUERY STYLE & DIFFICULTY ===

Cenderung formal (adversarial keren ditulis dengan diksi presisi tetapi misleading). Difficulty SEMUA `hard` by definition.
