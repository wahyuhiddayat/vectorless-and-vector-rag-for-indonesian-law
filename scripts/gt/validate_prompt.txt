=== ATURAN VALIDASI ===

HARD REJECT — tolak item ini, jangan masukkan ke benchmark:

1. CROSS_REF_ANCHOR
   answer_hint HANYA berisi referensi ke Pasal/ayat lain (contoh: "Pasal 4 ayat (3), ayat (4), ayat (5)").
   Artinya anchor node tidak punya konten substantif — jawabannya sebenarnya ada di node lain.

2. REFERENTIAL_QUERY
   Query mengandung frase yang merujuk ke konteks sebelumnya tanpa antecedent eksplisit, contoh:
   "aturan ini", "ketentuan tersebut", "hal itu", "yang tadi", "juga berlaku nggak".
   Query harus self-contained — bisa dipahami tanpa konteks apapun.
   CATATAN: "yang dimaksud" BUKAN trigger jika diikuti noun eksplisit dalam query yang sama.
     REJECT:  "Apa yang dimaksud?" / "Ketentuan yang dimaksud tersebut berlaku kapan?"
     ACCEPT:  "Apa yang dimaksud dengan penyadapan menurut UU KPK?"

3. MULTI_HOP
   Menjawab query secara lengkap membutuhkan informasi dari >1 leaf node yang berbeda.
   Benchmark ini single-hop only.
   Jangan flag MULTI_HOP kecuali ada bukti kuat dari konten anchor node atau answer_hint
   bahwa jawaban harus dikombinasikan dari ≥2 node berbeda (misalnya answer_hint secara eksplisit
   merujuk ke dua lokasi berbeda, atau anchor node hanya menyebut "lihat juga Pasal X" tanpa substansi).

SOFT FLAG — catat di flagged[], DAN lakukan aksi perbaikan pada cleaned array:

4. LEXICAL_LEAK
   Berlaku HANYA untuk item query_style = "colloquial".
   Query mengandung frasa 3+ kata berurutan yang sama persis dengan answer_hint.
   → AKSI di cleaned array: Tulis ulang query dengan sinonim/parafrase agar tidak ada overlap literal.
     Jika tidak bisa diperbaiki tanpa mengubah makna, HAPUS item dari cleaned array.

5. HARD_NO_SIBLING
   Berlaku untuk item difficulty = "hard".
   Berdasarkan navigation_path, tidak terlihat sibling leaf node di pasal yang sama
   yang bisa menjawab query secara plausibel tapi salah.
   → AKSI di cleaned array: Ubah "difficulty" dari "hard" menjadi "medium". Jangan hapus item.

6. VAGUE_COLLOQUIAL
   Query colloquial terlalu generik sehingga bisa cocok ke banyak node dalam dokumen
   (contoh: "Apa syaratnya?", "Bagaimana prosedurnya?", "Siapa yang bertanggung jawab?").
   → AKSI di cleaned array: HAPUS item dari cleaned array.

=== CATATAN ===
- Duplikat gold_anchor_node_id sudah dicek otomatis oleh validator kode — tidak perlu dicek di sini.
- Fokus hanya pada 6 aturan di atas.
- Jika item ACCEPT semua, cukup tulis summary saja tanpa list per-item.
- Jika ada baris [WARN] dari validator kode di bawah, gunakan sebagai PETUNJUK TAMBAHAN:
  * "[WARN] ... Anchor ayat text cites other provisions" → item ini kandidat CROSS_REF_ANCHOR
  * "[WARN] ... Possible context-dependent phrasing" → item ini kandidat REFERENTIAL_QUERY
  Konfirmasi tetap perlu dilakukan berdasarkan isi query dan answer_hint yang sebenarnya.
